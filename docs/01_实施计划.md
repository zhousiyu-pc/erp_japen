# 日本跨境电商 ERP 系统 - 分步实施计划

> 依据《日本跨境电商_ERP_系统_20260306_165701.md》详细需求文档

## 一、技术栈

| 层级 | 技术选型 | 说明 |
|------|----------|------|
| 前端 | Vue 3 + Vite + Element Plus | 已搭建，需按文档完善各模块 |
| 后端 | Spring Cloud Alibaba | 微服务架构 |
| 中间件 | RabbitMQ | 消息队列、异步任务 |
| 数据库 | MySQL 8.x | 主业务库 |
| 缓存 | Redis | 会话、权限、热点数据 |
| 搜索引擎 | Elasticsearch | 商品、订单全文检索（可选） |

## 二、三步实施路线

### 第一步：数据库层（当前）
1. 创建独立数据库 `erp_jp_crossborder`
2. 创建专用数据库用户及权限
3. 按需求文档生成核心表 DDL（租户、组织、用户、店铺、商品、订单等）

### 第二步：后端层
1. 初始化 Spring Cloud 父工程
2. 搭建网关、认证、公共模块
3. 按模块拆分微服务（system、product、order、wms、finance 等）
4. 集成 RabbitMQ、Redis、MyBatis-Plus
5. 实现多租户数据隔离、权限拦截

### 第三步：前端层
1. 按领星风格优化 UI 规范
2. 对接后端 API，替换 Mock 数据
3. 完善各模块表单校验、错误处理
4. 实现登录、权限、多租户切换

```bash
mysql -u root -p < 01_create_database_and_user.sql
mysql -u root -p < 02_schema_system.sql
mysql -u root -p < 03_schema_store.sql
```

```bash
cd backend/erp-store && mvn spring-boot:run
```

## 五、数据库设计原则（来自需求文档）

- **多租户隔离**：所有业务表含 `tenant_id` 字段，索引优先
- **逻辑删除**：使用 `is_deleted` 或 `del_flag`，不物理删除
- **审计字段**：`create_by`, `create_time`, `update_by`, `update_time`
- **日本本地化**：时区 JST，币种 JPY，日文字段支持
