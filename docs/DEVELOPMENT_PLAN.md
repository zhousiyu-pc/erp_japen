## ⚠️ 风险点

1. **乐天 API 申请** - 可能需要审核时间（当前使用模拟模式）
2. **OAuth 授权流程** - 需要配置回调域名
3. **数据一致性** - 同步失败需要重试机制
4. **API 限流** - 需要处理频率限制
5. **日本税务合规** - JCT 发票制度复杂

## 📝 开发规范

### Git 提交规范
```
feat: 新功能
fix: Bug 修复
docs: 文档更新
style: 代码格式
refactor: 重构
test: 测试相关
chore: 构建/工具
perf: 性能优化
```

### 代码规范
- 遵循阿里巴巴 Java 开发手册
- 使用 Checkstyle + SpotBugs 代码检查
- 统一异常处理
- 统一日志格式

### 测试规范
- 单元测试：JUnit 5 + Mockito
- 集成测试：TestContainers
- API 测试：Postman / Swagger
