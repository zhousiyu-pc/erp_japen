# 平台可插拔设计说明

## 一、设计目标

- **当前优先**：乐天市场（日本）平台对接
- **后续扩展**：亚马逊日本、雅虎购物等
- **架构原则**：可插拔、易扩展，新增平台无需修改核心业务代码

## 二、模块结构

```
erp-platform-api          # 平台接口定义（所有业务依赖此模块）
├── PlatformCode          # 平台编码枚举
├── PlatformAdapter       # 平台适配器接口
├── PlatformRegistry      # 适配器注册表（按 platformCode 获取）
├── PlatformCapability    # 平台能力声明
└── dto/auth/order/product/inventory  # 统一 DTO

erp-platform-rakuten      # 乐天市场实现（可插拔模块）
├── RakutenPlatformAdapter
└── （后续）RakutenApiClient、RakutenDto 等

erp-platform-amazon-jp    # 预留：亚马逊日本
erp-platform-yahoo-jp     # 预留：雅虎购物
```

## 三、使用方式

### 业务层调用示例

```java
@Autowired
private PlatformRegistry platformRegistry;

// 按 platform_code 获取适配器
PlatformAdapter adapter = platformRegistry.requireAdapter(PlatformCode.RAKUTEN);

// 或按字符串
PlatformAdapter adapter = platformRegistry.getAdapter("RAKUTEN");

// 调用统一接口，无需关心具体平台
PageResult<PlatformProduct> products = adapter.listProducts(storeId, query);
```

### 新增平台步骤

1. 在 `PlatformCode` 中新增枚举值
2. 新建模块 `erp-platform-xxx`，实现 `PlatformAdapter`
3. 使用 `@Component` 注册，Spring 自动注入 `PlatformRegistry`
4. 在需要使用的服务（如 erp-system）的 pom 中增加依赖

### 可选启用

通过配置关闭某平台：

```yaml
erp:
  platform:
    rakuten:
      enabled: false   # 不加载乐天适配器
```

## 四、与数据库对应

| 表字段 | 说明 |
|--------|------|
| t_store_master.platform_code | 与 PlatformCode 枚举值对应 |
| RAKUTEN / AMAZON_JP / YAHOO_JP | 已定义 |

## 五、接口能力

| 能力 | 说明 |
|------|------|
| 授权 | getAuthUrl / exchangeToken / refreshToken |
| 商品 | listProducts / getProduct / createOrUpdateProduct |
| 订单 | listOrders / getOrder |
| 库存 | syncInventory |

各平台按能力实现，未实现的方法可抛 `UnsupportedOperationException`。
